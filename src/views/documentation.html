Principales Difficultés et Solutions

Difficulté n°1 : Le cycle de vie du bouton d'installation PWA

Le problème : Notre bouton d'installation était intégré dynamiquement dans un header chargé via fetch() (Asynchrone). Le script de la PWA tentait d'attacher un événement 'click' sur ce bouton avant même qu'il n'existe dans le DOM, provoquant une erreur (null).

La solution apportée : Nous avons implémenté le principe de Délégation d'événements. Plutôt que de cibler le bouton, nous écoutons les clics sur document.body et vérifions si l'élément cliqué possède l'ID du bouton. Nous avons également ajouté une vérification via CSS (display-mode: standalone) pour cacher définitivement le bouton si l'application est déjà installée.

Difficulté n°2 : Gestion asynchrone des composants (Header/Footer) et des Chemins

Le problème : En modularisant le code (fichiers de config, layout.js), les chemins relatifs vers nos fichiers HTML ou JSON cassaient selon l'environnement d'exécution (serveur local vs machine d'un collègue), empêchant l'affichage de la page et la récupération des données.

La solution apportée : Nous avons centralisé tous les chemins dans une constante globale APP_CONFIG en utilisant des chemins relatifs stricts (./ au lieu de /) pour garantir que l'application s'exécute correctement depuis le dossier courant, peu importe la racine du serveur web utilisé.

Rétrospective : Ce que nous changerions

Si nous devions recommencer ce projet aujourd'hui, voici ce que nous ferions différemment :

  • L'architecture initiale : Nous mettrions en place la structure modulaire (les imports/exports, le fichier de configuration) dès le premier jour, plutôt que de refactoriser un gros fichier script.js à mi-parcours. Cela nous aurait évité beaucoup de bugs de chemins.
  • Faire des commits plus explicites pour la revue de code.
